第一部分：产品需求文档 (PRD)
项目名称： RealTalk - 职场英语实战 (MVP) 版本： V1.0
核心理念：
打造一个“在真实对话流中即学即用”的英语学习产品，帮助用户跨越“知道单词但不会应用”的鸿沟，专注于提升职场场景下的口语实战能力
1. 产品概述
1.1 背景与痛点
目标用户（大学生及职场新人）普遍面临：
“哑巴英语”：拥有一定词汇量，但在具体职场场景中无法组织有效、地道的英文表达。
“语境缺失”：传统学习方式脱离真实业务语境，不知道专业场景下的习惯用语（例如，在物流场景中用 on-time delivery 而非简单的 timely）。
“反馈缺失”：缺少低成本、即时、可反复的实战练习与反馈环境。
1.2 解决方案
通过 AI驱动的A/B角色扮演对话 模拟真实职场情境，采用 “提示 -> 思考 -> 表达 -> 翻牌核对” 的沉浸式自测学习模式，让用户在完成对话任务的过程中，主动构建语言能力。

2. 用户流程 (User Flow)
用户进入产品后，有三种学习路径（按核心优先级排列）：

- **路径 1（定向分难度练习）**：[核心] 直接抽取语料库中地道的 XHS 职场对话，零延迟开启实战。
- **路径 2（AI 口语搭子）**：进行非结构化的自由口语交流，锻炼即兴反应。
- **路径 3（自由场景练习）**：用户输入特定主题（如：物流催单），AI 结合语料库种子生成专属定制场景。

流程图概述：
首页 -> 核心导航 (分难度) -> 直接开始 / 首页 -> 其他模式 (AI搭子/自由主题)
场景对话页 -> 按照 “AI说 -> 用户答” 的回合制进行。
每个用户回合遵循：查看中文提示 -> 思考并组织英文 -> 录音回答 -> 提交评估 -> “翻牌”核对AI参考回答。
单场景结束后，可点击 “下一题” 重新生成新场景。

【更新：核心流程逻辑同步】
Step 3: 智能反馈与核对 (Feedback & Check)
1. 用户录音并确认转录。
2. 系统调用 AI 智能点评 (AI Evaluation)。
3. 返回评分、纠错及**综合反馈 (feedback)**。
4. 用户看完点评后，再点击 “查看参考答案” 展开地道表达。

3. 功能需求详情
3.1 首页
需求详情：
模式A入口（随机探索）： 醒目展示三个难度卡片：【初级 Beginner】、【进阶 Intermediate】、【高阶 Advanced】。
模式B入口（定向练习）： 顶部输入框。
模式C入口（AI口语对练）：【更新】新增自由对话卡片，入口为 `/freetalk`。

3.2 场景对话模块（核心学习界面）
交互逻辑：
【更新】后端一次性生成 4-6 轮的完整对话 JSON，前端负责状态流转。
AI回合：显示英文和中文翻译。
用户回合：
- 提示：显示 reference.answer 的逐字翻译。
- 录音：通过浏览器 API 进行语音识别。
- 评分：提交请求至 `/api/answers/evaluate`。

3.3 后端场景引擎
- **分难度练习**：直接从 BigQuery 检索 `example_json`，无需 AI 二次生成，追求极致响应。
- **自由场景练习**：统一使用 `qwen3-max` 模型，结合 BQ 语料种子与用户自定义 Topic 进行综合生成。
- **数据源**：Google BigQuery (`xhs_structured_corpus`)。

3.4 全局功能
“下一题”功能：
逻辑：点击后前端重新调用 `/api/scenarios/random` 或 `/api/scenarios/custom`，本地刷新场景状态。【更新：移除了单步 Next API，改为全场景刷新】

后端 API 列表（最新同步版本）
1) 获取练习场景
GET /api/scenarios/random?level=beginner|intermediate|advanced
返回：场景标题、描述、完整消息列表（包含 AI 回合与 User 提示/参考答案）

GET /api/scenarios/custom?topic=...&level=...
返回：同上结构

2) 用户回答评估
POST /api/answers/evaluate
入参： { "prompt": "中文提示", "userAnswer": "用户英语回答" }
返回：【更新：包含字段对齐】
{
  "score": 85, // 0-100 整数
  "suggestedAnswer": "改进后的英文答案",
  "keyPhrases": ["关键短语1", "关键短语2"],
  "corrections": ["错误1 -> 正确1"],
  "feedback": "综合评估建议（中文）"
}

3) 语音功能（已实现）
POST /api/stt -> 语音转文字
POST /api/tts -> 文字转语音（可选，当前前端使用内置合成或 API 调用）
